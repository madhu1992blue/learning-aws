# Diagrams

## Group Icons

These are used as containers to group related resources together.
Ex:
- VPC
- Private Subnet
- Public Subnet
- Region
- AWS Account

## Category Icons

These are used to represent a category of services.
Ex:
- Compute
- Storage
- Database
- Networking

## Service Icons

The actual services in each Category are represented using Service Icons.
Ex:
- EC2 service under Compute category
- S3 service under Storage category
- RDS service under Database category
- VPC service under Networking category

## Resource Icons

Individual resources within each service are represented using Resource Icons.
Ex:
- An EC2 instance within the EC2 service
- An S3 bucket within the S3 service
- An RDS database instance within the RDS service
- An Internet Gateway or a Peering connection within the VPC service
